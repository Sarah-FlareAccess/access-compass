HUMAN REVIEW CHECKLIST
Using AI to Create Accessible and Inclusive Communications (Flare Access)

Use this checklist before you publish any draft you produced with help from AI. AI can draft well. It cannot verify facts about your business, walk your venue, or speak to your real audience. That is your job. Tick off each item before you publish.

============================================================
CHECKLIST
============================================================

[ ] Is everything factually correct about your business and venue?

[ ] Have you tested it with at least one person from the target audience? If not, plan to before publishing.

[ ] Have any photos or images been described accurately?

[ ] For Easy Read or Visual Communication: have you sourced images or symbols?

[ ] For Accessibility Guides: has someone walked the route and verified the details?

[ ] Have you removed any AI-added claims that you cannot verify?

[ ] Have you added the publishing date and a way for readers to give feedback?

============================================================
FEEDBACK PARAGRAPH TEMPLATE
============================================================

Use a real address that a real person reads. Add a version like this to every accessible piece you publish.

"Was this useful? We'd love to know if anything could be clearer, better translated, or missing. Email accessibility@[yourbusiness].com.au or tell us in person at reception. We update this content when we hear from you."

============================================================
PUBLISHING CHECKLIST FOR TAGGED PDFs
============================================================

Before exporting to PDF from Word:

[ ] All headings use Word's built-in Heading 1, Heading 2 etc. styles (not just bold text)
[ ] Every image has alt text set via Picture Format > Alt Text
[ ] Decorative images are marked as decorative
[ ] Links use descriptive text (not "click here")
[ ] Tables have a header row set and no merged cells
[ ] Document title, language and author are set in File > Info > Properties
[ ] Run Word's built-in Accessibility Checker (Review > Check Accessibility)
[ ] Export as PDF with "Document structure tags for accessibility" enabled

After export, if you have Adobe Acrobat Pro:

[ ] Run Tools > Prepare for Accessibility > Accessibility Check
[ ] Fix any flagged issues (missing tags, reading order, etc.)
[ ] Re-run the check

============================================================
PUBLISHING CHECKLIST FOR GOOGLE DOCS
============================================================

Before publishing or exporting from Google Docs:

[ ] All headings use Format > Paragraph styles (Heading 1, Heading 2 etc.), not just bold or larger text. The "Title" style is not a heading.
[ ] Heading order does not skip levels (check via View > Show outline)
[ ] Every image has alt text set via right-click image > Alt text
[ ] Links use descriptive text (not "click here"), set via Insert > Link
[ ] Tables use a single header row and no merged cells (Google Docs cannot tag a semantic header row, so keep tables simple)
[ ] Document title and language are set via File > Document details
[ ] Reading order is correct top to bottom

Known limits to work around:

[ ] Google Docs cannot mark images or tables as decorative. Remove any purely decorative image, or a screen reader will still announce it.
[ ] Google Docs has no built-in accessibility checker. Work through the content checklist above carefully before you publish.
[ ] For a tagged PDF, export to .docx and finish in Microsoft Word (verify heading styles, then export with "Document structure tags for accessibility" enabled). Google Docs PDF export does not reliably produce a tagged PDF.
